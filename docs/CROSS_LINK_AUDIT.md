## 1. Slug Function Comparison

Seven slug functions exist across the codebase. Three are canonical (in lib/);
four are inline clones in page/component files.

| Function | Location | Apostrophe strip | Notes |
|---|---|---|---|
| mobToSlug | lib/mob-slug.ts | No | Has explicit /-+/ collapse step |
| zoneToSlug | lib/zone-slug.ts | No | Has .trim() step |
| itemToSlug | lib/item-slug.ts | YES (canonical) | Strips apostrophes BEFORE normalising |
| itemSlug | lib/crafting.ts | No | Inline clone -- DIVERGES for apostrophe names |
| mobSlug | app/factions/page.tsx | No | Inline clone without /-+/ collapse |
| zoneSlug | app/factions/page.tsx | No | Inline clone without .trim() |
| toUrlSlug | app/epics/EpicTrackerClient.tsx | No | Inline clone -- DIVERGES for apostrophe names |

### Key differences

1. APOSTROPHE STRIPPING (CRITICAL): itemToSlug strips apostrophes before normalizing.
   Inline clones in lib/crafting.ts and EpicTrackerClient.tsx do NOT strip apostrophes.
   Apostrophes in those clones become hyphens, diverging from the canonical slug map.
   buildItemSlugMap uses itemToSlug, so inline-generated links to apostrophe-named
   items will be dead links at runtime. 102 items are affected.

2. HYPHEN COLLAPSE: mobToSlug has explicit /-+/g collapse; others rely on [^a-z0-9]+.
   Functionally equivalent. No output difference in practice.

3. TRIM: zoneToSlug calls .trim(); inline clones skip it. No practical impact since
   zone names in JSON data have no leading/trailing whitespace.

4. MOB PAGE ITEM LINKS (BUG): app/mob/[name]/page.tsx line ~189 calls mobToSlug(item)
   to generate /item/[slug] hrefs in the loot pool list. Should call itemToSlug(item).
   102 items with apostrophes are affected.

## 6. Recommendations

### Priority 1 -- Fix broken item links (102 items affected)

Fix A: app/mob/[name]/page.tsx around line 189
  The const itemSlug variable is built with mobToSlug(item).
  Import itemToSlug from @/lib/item-slug and call itemToSlug(item) instead.

Fix B: components/CraftingTabs.tsx
  The file imports itemSlug from @/lib/crafting.
  Add import { itemToSlug } from "@/lib/item-slug" and replace all itemSlug() calls.

### Priority 2 -- Eliminate inline slug clones (risk reduction)

Fix C: app/factions/page.tsx
  Remove local mobSlug and zoneSlug function definitions.
  Import mobToSlug from @/lib/mob-slug and zoneToSlug from @/lib/zone-slug instead.

Fix D: app/epics/EpicTrackerClient.tsx
  Replace toUrlSlug with targeted imports:
    Zone links: import { zoneToSlug } from "@/lib/zone-slug"
    Mob links:  import { mobToSlug } from "@/lib/mob-slug"

### Priority 3 -- Zone connections (low severity navigation gap)

Fix E: lib/zone-connections.ts
  Add reverse entries for the 12 one-directional connections listed in Section 5.
